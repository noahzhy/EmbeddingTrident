## Ray Core (`@ray.remote`) vs. Ray Serve (`@serve.deployment`)

| 特性         | Ray Core (`@ray.remote`)                      | Ray Serve (`@serve.deployment`)                             |
| :----------- | :-------------------------------------------- | :---------------------------------------------------------- |
| **定位**     | **分布式计算**。把函数或类扔到集群里跑。      | **模型/应用服务**。把类变成一个带接口的在线服务。           |
| **访问方式** | 通过 Python 代码里的 `handle.remote()` 调用。 | 通过 **HTTP 请求** 或 `DeploymentHandle` 调用。             |
| **扩缩容**   | 相对底层，通常需要手动管理 Actor 数量。       | **自动扩缩容 (Autoscaling)**。根据流量压力自动增减副本。    |
| **生命周期** | 任务完结或 Python 脚本退出通常就没了。        | **持久化服务**。部署后一直运行，直到手动 `serve shutdown`。 |
| **调度重点** | 追求极致的低延迟、复杂的分布式逻辑。          | 高可用性、流量切分、版本管理。                              |

---

### 核心差异点总结

* **Ray Core**：侧重于**任务并行**。适合大规模数据处理、分布式训练等“算完就结束”的场景。
* **Ray Serve**：侧重于**服务化**。适合将模型封装成 API 接口，处理来自 Web 的并发请求。
